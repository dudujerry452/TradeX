# 项目结构

```
TradeX/
├── backend/               # Django 后端
│   ├── manage.py
│   ├── tradeX/            # 项目配置 (settings, urls, asgi)
│   └── core/              # 业务逻辑
│       ├── api/           # API 模块 (按功能拆分)
│       │   ├── __init__.py          # 路由聚合
│       │   ├── common.py            # 共享 schemas 和认证
│       │   ├── auth.py              # 登录认证
│       │   ├── users.py             # 用户管理
│       │   ├── products.py          # 商品管理
│       │   ├── categories.py        # 分类管理
│       │   ├── tags.py              # 标签管理
│       │   ├── favorites.py         # 收藏管理
│       │   ├── recommendations.py   # 推荐系统
│       │   ├── orders.py            # 订单系统
│       │   ├── notifications.py     # 通知系统
│       │   ├── uploads.py           # 文件上传
│       │   ├── rag.py               # AI RAG
│       │   ├── chat.py              # 聊天 REST API
│       │   ├── forum.py             # 论坛 API
│       │   └── system.py            # 系统接口
│       ├── consumers.py   # WebSocket 消费者 (聊天)
│       ├── routing.py     # WebSocket 路由
│       ├── models.py      # 数据库模型
│       ├── tests.py       # 测试
│       └── management/commands/seed.py  # 测试数据
├── webview/               # Vue 3 + Vite 前端
├── mobile/                # Flutter 移动端
├── mini-program/          # uni-app 打包的小程序
├── script/                # 部署与开发脚本
│   ├── switch-config.sh   # 切换 deploy-config 环境
│   ├── run-backend.sh     # 启动后端 (HTTP + WebSocket)
│   ├── run-web.sh         # 启动前端
│   ├── start-dev.sh       # 一键启动开发环境 (双端口)
│   └── deploy.sh          # 部署脚本
├── deploy-config.json     # 统一环境配置
└── doc/                   # 文档
```

后端 API 默认运行在 `0.0.0.0:8000`，WebSocket 运行在 `0.0.0.0:8001`。
前端开发服务器运行在 `127.0.0.1:5173`，`/api/*` 请求由 Vite 代理到后端。

# 环境搭建

## 方式一：Nix（for dudujerry452）

```bash
nix develop   # 自动安装所有后端依赖
```

依赖版本由 `flake.nix` 锁定：Python 3.12、Django 6.0.3、django-ninja 1.6.0。

## 方式二：手动

require：**Python 3.12**、**Node.js 22+**、**pnpm/npm**、**Flutter**（移动端）。

**后端**

```bash
python3.12 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

pip install "django==6.0.3" "django-ninja==1.6.0" "django-cors-headers" "channels" "daphne"

# RAG 聊天能力
pip install "openai" "chromadb"

# 腾讯云 COS 对象存储
pip install -U cos-python-sdk-v5
```

在 `backend/.env.local`（或 `.env.development`、`.env.production`）中配置模型密钥：

```bash
OPENROUTER_API_KEY=xxxxxxxxxxxxxxxx
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
```

Django 通过 `DJANGO_ENV` 环境变量决定加载哪个 `.env` 文件：
- `local` → `.env.local`
- `development` → `.env.development`
- `production` → `.env.production`

**前端**

```bash
cd webview
pnpm install      # or npm install
```

如未安装 pnpm，用 `npm install -g pnpm` 安装

# 启动开发服务器

## 方式一：脚本一键启动（推荐）

先确保已生成 `deploy-config.json`：

```bash
./script/switch-config.sh local        # 切换为本地开发配置
# 或
./script/switch-config.sh development  # 切换为远程开发配置
```

然后启动开发环境（双端口：HTTP + WebSocket）：

```bash
./script/start-dev.sh
```

输出示例：

```
环境: local
Django HTTP:   http://localhost:7001
WebSocket:     ws://localhost:7002/ws/chat/
API 文档:       http://localhost:7001/api/docs
```

## 方式二：手动分步启动

**后端**

```bash
# 数据库迁移
cd backend
python manage.py migrate

# 启动 HTTP + WebSocket
../script/run-backend.sh 8000 8001
```

**前端**

```bash
cd webview
pnpm dev          # or npm run dev
```

浏览器访问 `http://127.0.0.1:5173`。

# 常用命令

| 操作 | 命令 |
|------|------|
| 数据库迁移 | `python backend/manage.py migrate` |
| 生成迁移文件 | `python backend/manage.py makemigrations` |
| 写入测试数据 | `python backend/manage.py seed` |
| 创建后台管理员 | `python backend/manage.py createsuperuser` |
| 运行测试 | `python backend/manage.py test core` |
| 检查配置 | `python backend/manage.py check` |
| 构建前端 | `cd webview && pnpm build` |
| 同步向量库 | `python backend/manage.py sync_vector_products` |
| 切换环境配置 | `./script/switch-config.sh <local|development|production>` |

# 测试数据（seed）

`seed` 命令向数据库写入一套完整的前端测试数据，可重复运行，不会重复插入。

```bash
python backend/manage.py seed
```

写入内容：

| 数据 | 说明 |
|------|------|
| 用户 × 5 | admin_root（管理员）、buyer_alice、buyer_bob（买家）、seller_techshop、seller_newbie（卖家，待审核）|
| 商品 × 4 | 2 已上架、1 待审核、1 已下架 |
| 分类 × 4 | 数码/家居/服饰/其他 |
| 标签 × 8 | 各类商品标签 |
| 注册审核 × 3 | 3 条通过记录 |
| 商品审核 × 2 | 2 条通过记录 |
| 订单 × 2 | 1 已完成、1 已出货 |
| 订单明细 × 2 | 每个订单各一条 |
| 订单日志 × 3 | 订单操作记录 |
| 通知 × 2 | 系统通知 |
| 留言 × 2 | 1 条已回复、1 条未回复 |

所有测试数据的 ID 以 `seed_` 开头，与业务数据区分。

`sync_vector_products` 命令将商品数据库中的数据同步到向量库中：

```bash
python backend/manage.py sync_vector_products
```
