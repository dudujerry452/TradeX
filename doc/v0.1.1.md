# TradeX v0.1.1 版本更新说明

## 概述

本版本主要完成商品分类体系的重构，从简单的字符串分类升级为独立的分级管理模型，同时优化移动端分类筛选体验。

---

## 后端变更

### 1. 新增 Category 分类模型

**文件**: `backend/core/models.py`

新增独立的商品分类表，支持分类的增删改查管理。

```python
class Category(models.Model):
    category_id = models.CharField(max_length=50, primary_key=True)
    name = models.CharField(max_length=100)
    description = models.TextField(blank=True)
    sort_order = models.IntegerField(default=0)
    is_active = models.BooleanField(default=True)
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
```

**特性**:
- 支持分类启用/禁用（软删除）
- 支持自定义排序（sort_order）
- 支持分类描述

### 2. Product 模型扩展

**文件**: `backend/core/models.py`

新增 `category_ref` 外键字段，同时保留旧 `category` 字段兼容。

```python
class Product(models.Model):
    # 保留旧字段兼容
    category = models.CharField(max_length=100, blank=True, verbose_name="商品分类(旧)")
    # 新增外键关联
    category_ref = models.ForeignKey(
        Category,
        on_delete=models.SET_NULL,
        null=True,
        blank=True,
        related_name='products',
        verbose_name="所属分类"
    )
```

### 3. 新增分类管理接口

**文件**: `backend/core/api.py`

| 接口 | 方法 | 说明 | 权限 |
|------|------|------|------|
| `/api/categories/` | GET | 获取分类列表 | 无限制 |
| `/api/categories/` | POST | 创建分类 | 需认证 |
| `/api/categories/{id}/` | GET | 获取分类详情 | 无限制 |
| `/api/categories/{id}/` | PUT | 更新分类 | 需认证 |
| `/api/categories/{id}/` | DELETE | 禁用分类（软删除） | 需认证 |

**响应示例**:
```json
{
    "category_id": "misc",
    "name": "杂物",
    "description": "各种杂物",
    "sort_order": 1,
    "is_active": true
}
```

### 4. 搜索接口增强

**文件**: `backend/core/api.py`

`GET /api/products/search/` 新增 `category` 查询参数：
- `q` 为空 + `category` 有值：返回该分类下所有商品
- `q` 有值 + `category` 有值：在该分类内搜索
- 同时兼容新旧分类字段查询

**请求示例**:
```
GET /api/products/search/?category=misc&limit=10&offset=0
```

### 5. 商品创建接口适配

**文件**: `backend/core/api.py`

`POST /api/products/` 支持新的分类外键：
- 自动根据传入的 `category` 值查找对应的 Category 对象
- 同时填充新旧字段，确保兼容性

### 6. 数据库迁移

**迁移文件**:
- `0003_category_product_category_ref.py` - 创建 Category 表和 Product 外键
- `0004_init_categories.py` - 初始化分类数据并迁移旧数据

**初始化分类**:
| ID | 名称 | 说明 |
|----|------|------|
| misc | 杂物 | 各种杂物 |
| digital | 数码 | 数码产品 |
| books | 旧书 | 二手书籍 |
| other | 其他 | 未分类商品（兜底） |

**旧数据迁移**:
- 所有无 `category_ref` 的商品自动归入 "other" 分类
- 保留原 `category` 字段值

## 移动端变更

### 1. API 层适配

**文件**: `mobile/lib/api.dart`

**修改**:
- `getCategories()` - 改为从后端 `/api/categories/` 动态获取
- `searchProducts()` - 新增可选 `category` 参数

### 2. 创建商品页面

**文件**: `mobile/lib/create_product_page.dart`

**修改**:
- 分类下拉框改为动态加载（`initState` 时调用 `_loadCategories()`）
- 移除硬编码分类列表
- 修复分类加载时的字体显示异常

### 3. 发现页面

**文件**: `mobile/lib/discover_page.dart`

**修改**:
- "最新" Tab 使用搜索 API 进行后端分类筛选
- 推荐 Tab 选择特定分类时切换到搜索 API
- 修复上滑加载更多时分类筛选失效的 Bug
- 新增 `_loadLatestProducts()` 和 `_loadMoreLatest()` 方法

**加载逻辑**:
| Tab | 分类 = "全部" | 分类 != "全部" |
|-----|--------------|----------------|
| 推荐 | 个性化/热门推荐 | 搜索 API（带分类筛选） |
| 最新 | 搜索 API（无分类） | 搜索 API（带分类筛选） |

## Bug 修复

### 1. 分类字体异常

**问题**: 分类下拉框加载时显示异常字体

**修复**: 加载中时不显示选项（`items: []`），使用 `disabledHint` 显示"加载中..."

### 2. 加载更多分类失效

**问题**: 选择分类后上滑加载更多，又调用了推荐 API

**修复**: 根据 Tab 和分类状态决定调用哪个加载方法

### 3. 旧数据搜索不到

**问题**: 旧商品的 `category` 字段存的是中文，前端传的是英文 ID

**修复**: 搜索接口同时查询新旧字段，并支持按分类名称匹配

## API 更新汇总

### 新增接口

#### 分类管理

**GET /api/categories/**
```json
// Response
[
    {
        "category_id": "misc",
        "name": "杂物",
        "description": "各种杂物",
        "sort_order": 1,
        "is_active": true
    }
]
```

**POST /api/categories/**
```json
// Request
{
    "name": "服饰",
    "description": "服装鞋帽",
    "sort_order": 4
}
```

#### 商品状态更新

**PATCH /api/products/{product_id}/status/**
```json
// Request
{
    "product_status": "OFF_SHELF"
}

// Response
{
    "product_id": "xxx",
    "product_status": "OFF_SHELF",
    "message": "商品状态已更新为: 已下架"
}
```

示例: `GET /api/products/search/?category=misc`
