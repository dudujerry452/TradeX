选题: 

4、商品网上交易系统要求

（1）用户管理：主要包括注册、登录、留言等功能模块。

（2）商品管理：主要包括发布在线产品展示功能（必须有图片的形式）以及对商品信息的管理。

（3）审查管理：主要包括管理员对注册的审查以及商品的审查。

（4）用户购物：主要包括用户能对已经审核通过的商品进行在线产品查找功能，并且进行在线下订单购物功能。

（5）配送：主要包括买家和卖家对订单的一系列操作：用户下订单后。卖家在发现后修改订单状态为出货，买家收到物品后能将订单状态修改为已收货，交易成功结束。

## Q&A: 

**用什么过程模型? ** 增量模型, 容易得高分. 

暂且分为以下阶段: 

- v1.0 基本功能
- v1.0 前端功能
- v1.0 完整功能

每次增量都要进行分析, 开发和测试. 

**怎么写增量? 在哪里体现? **

- 02 - 软件开发计划
- 19 - 开发进度月报
- 22 - 软件版本说明
- 21 - 软件规格说明
- qt - 软件需求变更单(需要PM制造不同阶段的需求)

**我们的计划是什么**?

- 每两周推进一次大版本
- 确保所有人有足够的commit展示在git记录中

**怎么写文档?**

- 我们用GB T-8567-2006计算机软件文档编制规范, 也就是群里发的那个压缩包的规范. 对于超出作业范围的标注[deprecated], 最后统一删除
- 用Markdown书写, 提交到仓库doc分支
- 规范中设计的文件(带编号的)提交到doc分支的GBT8567文件夹. 其他文件存储位置暂定(建议在群中讨论)

**git有说法吗?**

- 请阅读doc分支的 git规范.md 文件. 
- 避免繁琐的命令操作, 建议使用Fork/Lazygit等git工具. 
- 建议积极提交commit, 若每个周都有均匀的commit, 可体现各组员成果和增量开发模型. 

**我们希望什么?**

- 得到高分
- 打造一个实用, 鲁棒的交易系统
- 给每个成员带来一次有意义的实践经验
