## 1 引言

### 1.1 标识
* **项目名称**：商品网上交易系统
* **文档标识**：可行性分析报告（FAR）- 初稿

### 1.2 背景
本项目是在软件工程课程实践要求下提出的模拟商业项目  。旨在开发一个能够支持用户在线注册、发布商品、审核商品以及完成在线交易全流程的 C2C/B2C 电商平台。提出者的核心要求和目标是通过团队协作，在有限周期内高效实现一套功能完整的标准 Web 业务系统 。

### 1.4 文档概述
 本文档旨在对“商品网上交易系统”进行初期的可行性分析，从技术、经济和法律等方面论证项目的可行性，作为后续项目立项和系统开发的决策依据   。

## 3 可行性分析的前提

### 3.1 项目的要求
 本项目必须实现以下五大核心模块 ：
1. **用户管理**：主要包括注册、登录、留言等功能模块。
2. **商品管理**：主要包括发布在线产品展示功能（必须有图片的形式）以及对商品信息的管理。
3. **审查管理**：主要包括管理员对注册的审查以及商品的审查。
4. **用户购物**：主要包括用户能对已经审核通过的商品进行在线产品查找功能，并且进行在线下订单购物功能。
5. **配送**：主要包括买家和卖家对订单的一系列操作：用户下订单后，卖家在发现后修改订单状态为出货，买家收到物品后能将订单状态修改为已收货，交易成功结束。

### 3.2 项目的目标
 在课程规定的时间内，以最低的学习成本和最高的开发效率，交付一个界面友好、业务逻辑闭环、无明显重大 Bug 的在线交易系统 。

### 3.3 项目的环境、条件、假定和限制
*  **环境限制**：团队共 4 人（兼顾产品、前后端、测试），开发周期较短 。
*  **资金限制**：项目无实际商业预算，需全部采用开源免费的技术和工具 。

### 3.4 进行可行性分析的方法
 通过对比不同的技术栈方案，结合现有团队成员的技术储备、时间成本，以及开源社区的支持力度，进行综合评估与论证 。

---

## 4 可选的方案

### 4.3 可选择的系统方案1
*  **方案 1（Go + Vue 前后端分离）**：运行效率极高，但对于 4 人学生团队来说，需要手写大量基础后台代码与鉴权逻辑，开发周期长，风险高 。

### 4.4 可选择的系统方案2
*  **方案 2（Python Django + Bootstrap 原生模板）**：开发速度极快，自带完善的后台管理员审查系统，对表单验证和图片上传支持极好，能最大程度缩短开发时间 。

### 4.5 选择最终方案的准则
 以“开发效率最高、学习成本最低、能完美覆盖核心需求”为准则，最终选择 **方案 2（Python Django 技术栈）** 。

---

## 5 所建议的系统

### 5.1 对所建议的系统的说明
 采用基于 Python 的 Django Web 框架开发的 B/S 架构系统  。系统自带的 Admin 模块直接用于处理“审查管理”需求；使用 Django 原生模板结合 Bootstrap 框架构建前端用户界面 。

### 5.2 数据流程和处理流程
*  **注册与发布**：用户注册 -> 等待管理员审查 -> 审查通过 -> 用户发布带图片的商品 -> 等待管理员审查 -> 商品上架展示 。
*  **交易与配送**：买家搜索商品 -> 下单 -> 卖家后台查收订单 -> 卖家修改状态为“已出货” -> 买家确认收货 -> 状态修改为“已收货” -> 交易闭环 。

### 5.4 影响(或要求)

#### 5.4.1 设备
 开发人员需配备普通 PC 电脑即可 。

#### 5.4.2 软件
 操作系统（Windows/Mac），开发语言（Python 3.x），框架（Django），数据库（SQLite/MySQL），前端（HTML/CSS/Bootstrap） 。

#### 5.4.6 经费
 所有软件环境均为开源免费，部署测试可使用本地 localhost，经费为 0 。

## 6 经济可行性(成本----效益分析)

### 6.1 投资
 本项目为教学实践项目，不涉及实际基本建设投资（硬件采购与人员工资薪酬），主要投资为团队 4 名成员的时间成本和精力投入 。

### 6.2 预期的经济效益
 虽然无直接商业现金收益，但其“收益”体现在：团队成员获得完整的软件工程实操经验，以及最终提交作业所带来的学分收益 [cite: 3] 。收益远大于时间成本投入，经济上完全可行 [cite: 3]。

---

## 7 技术可行性(技术风险评价)
 本团队现有 4 名成员合理分工（产品兼甲方、前端、后端、测试）。由于选择了封装度极高的 Django 框架，极大地降低了系统底层的开发难度。框架自带的用户系统和 Admin 管理系统完美契合项目的“用户管理”和“审查管理”需求。综合人员、技术条件评估，技术层面上风险极低，具备充分的技术可行性 [cite: 3]。

---

## 8 法律可行性
 本项目采用的所有技术栈（Python, Django, Bootstrap 等）均遵循开源协议。所使用的测试图片和数据均为网络公开且非商业用途。系统开发不存在侵犯专利权、版权等违法风险和责任，法律上完全可行 [cite: 3]。

---

## 9 用户使用可行性
 系统采用标准的电商网站交互逻辑（如购物车、订单状态流转等），用户无需经过专门培训即可熟练使用浏览器进行注册、购物和发货操作，符合用户使用习惯，使用可行性高 [cite: 3]。
