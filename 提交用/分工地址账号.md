# 总分工

1. **PM / 甲方 (黄坤)**：
   * **工程职责**：负责拆解电商需求为 User Story，管理产品待办列表 Backlog，把控整体项目进度与最终产品验收。
   * **文档负责**：主笔 `01-可行性分析(FAR)`、`02-软件开发计划(SDP)`（包含绘制甘特图）、`06-运行概念说明(OCD)`、`07-系统需求规格说明(SSS) / 11-软件需求规格说明(SRS)`（绘制大量用例图与业务流程图）、`19-开发进度月报(DPMR)` 以及期末撰写`20-项目开发总结报告(PDSR)`。
2. **统筹者 (李子豪)**：
   * **工程职责**：负责制定 Git 协作流；管理需求审批与代码修改权限；主导团队 Code Review；设计各类系统测试用例。
   * **文档负责**：主笔 `12-数据需求说明(DRD)`、`17-软件配置管理计划(SCMP)`、`18-软件质量保证计划(SQAP)`、`03-软件测试计划(STP) / 15-软件测试说明(STD)`（设计输入错误密码等具体测试用例），并负责编制开发过程中的 `软件问题报告` 与 `软件需求变更单`。
3. **主程 / 架构师 (周致远)**：
   * **工程职责**：负责整体系统前后端架构设计、数据库表结构设计与调优；主导 Django 后端业务逻辑开发与 API 设计。
   * **文档负责**：主笔`09-系统设计说明(SSDD) / 13-软件设计说明(SDD)`（绘制系统架构图）、`14-数据库设计说明(DBDD)`（详细记录表结构）、`08-接口需求说明(IRS) / 10-接口设计说明(IDD)`（依托 Swagger/Apifox 导出），以及 `25-计算机编程手册(CPM)`（制定代码规范）。
4. **前端 / 测试运维 (刘俊涵)**：
   * **工程职责**：负责 Web 端及移动端界面的开发；依据统筹者的计划执行测试；主导 Python 后端环境的部署上线、基础运维。
   * **文档负责**：主笔`16-软件测试报告(STR)`（依据计划执行测试并填入截图）、`23-软件用户手册(SUM)`（图文并茂指导用户/老师操作）、`24-计算机操作手册(COM)`、`04-软件安装计划(SIP) / 05-软件移交计划(STrP)`（撰写如何部署 Python 环境与 runserver 指南），以及 `21-产品规格说明(SPS) / 22-软件版本说明(SVD)`。
