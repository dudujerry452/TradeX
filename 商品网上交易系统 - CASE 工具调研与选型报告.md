
# 商品网上交易系统 - CASE 工具调研与选型报告

## 1 调研背景与目标
在“商品网上交易系统”的开发周期中，为了提高团队协作效率、规范系统建模、并配合我们采用的 CI/CD 自动化流水线，本小组对目前主流的 CASE（计算机辅助软件工程）工具进行了系统性调研。
我们的核心选型诉求是：**跨平台（全兼容 Windows, macOS, Linux）、轻量敏捷、且生成的设计产物必须能被 Git 完美追踪**

## 2 主流 CASE 工具调研分析
根据业界应用情况，我们将调研的 CASE 工具分为三大类进行分析：

### 2.1 传统重型 CASE 工具（系统建模与架构）
这类工具功能极其强大，通常提供从业务分析到代码双向工程的完整解决方案。
* **Rational Rose / Sybase PowerDesigner**
    * **用途**：企业级 UML 建模、数据库 ER 图设计、复杂架构设计。
    * **技术特点**：重资产、重流程，通常与特定的技术栈深度绑定。
    * **淘汰理由**：过于笨重，且产出文件为专有的二进制格式，Git 无法进行行级别的变更追踪，且对 macOS/Linux 支持极差。
* **Microsoft Visio**
    * **用途**：通用的流程图、网络拓扑图绘制。
    * **技术特点**：界面友好，Office 生态融合度高。
    * **淘汰理由**：仅限 Windows 平台，且 `.vsdx` 格式不支持 Git 差异比对。

### 2.2 现代独立 CASE 工具（敏捷建模）
这类工具是目前中小型团队画图的主力。
* **Enterprise Architect / Visual Paradigm**
    * **用途**：专业的敏捷 UML 建模，支持需求追踪。
    * **技术特点**：支持多平台，图表美观，支持导出多种文档。
    * **淘汰理由**：商业收费软件，且工程文件依然是以 XML 或私有格式打包，Git 追踪体验不佳，容易在多人协作时产生合并冲突。
* **StarUML**
    * **用途**：轻量级、跨平台的 UML 绘图工具。
    * **技术特点**：界面现代化，响应快。但其底层工程数据依然不适合纯文本式的版本控制。

### 2.3 代码驱动型 / 持续集成 CASE 工具（我们的重点）
* **PlantUML**
    * **用途**：通过编写**纯文本代码**来生成 UML 图表（时序图、用例图、类图等）。
    * **技术特点**：完全开源，跨平台。**它是“架构即代码 (Architecture as Code)”的最佳实践。**
* **Git**
    * **用途**：分布式源代码版本控制工具（Lower CASE）。
    * **技术特点**：目前全球软件开发的绝对基建，支持分支管理与多人协同。
* **Apache JMeter**
    * **用途**：软件压力测试、接口功能测试工具。
    * **技术特点**：纯 Java 编写，天生跨平台。测试脚本保存为 XML/JMX 文本格式，可纳入 Git 管理。

## 3 小组 CASE 工具最终选型及理由

结合本项目“Django + Vue/Flutter”的前后端分离技术路线，以及 4 人团队跨设备开发的现状，本小组决定采用以下 CASE 工具组合：

### 3.1 核心建模工具：PlantUML (Upper CASE)
* **入选理由：**
    1.  **完美契合 Git 追踪**：PlantUML 使用纯文本编写图表逻辑（如 `Lizihao -> Zhouzhiyuan: 发起支付请求`），Git 可以极其精准地记录每次架构设计的修改历史，彻底告别团队协作时的“二进制文件冲突”。
    2.  **绝对的跨平台**：团队成员只需在 VS Code 中安装 PlantUML 插件，无论使用 Mac 还是 Windows，都能实时预览生成的架构图。
    3.  **完美融入 CI/CD**：我们的设计文档（Markdown）内嵌 PlantUML 代码后，GitHub/GitLab 均原生支持渲染，无需在仓库中保存臃肿的图片文件。

### 3.2 版本控制与测试工具组合 (Lower CASE)
* **Git**：作为底层的版本控制工具，承载代码、PlantUML 设计图以及测试用例的管理。
* **Apache JMeter**：针对本项目的“商品搜索”与“在线下单”高频并发接口，采用跨平台的 JMeter 编写测试计划，确保系统交付质量。

**结论：** 选用 **PlantUML + Git + JMeter** 的 CASE 工具链，不仅实现了 100% 跨平台与免费开源，更是将“系统建模”也纳入了 Git 的代码化管理体系，极大地提升了敏捷开发的工程规范性。
